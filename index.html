📊 Project Overview

🎮 Project Goals

Create a 2D Mount & Blade inspired RPG with:

  • ⚔️ Tactical directional combat system
  • ⚔️ Character progression (XP, stats, skills)
  • ⚔️ Faction and reputation mechanics
  • Party/companion recruitment
  • Open world exploration
  • Pixel art graphics (16x16 or 32x32)

🛠️ Tech Stack

Language: Python 3.12 (Pygame compatible)

Language: Python 3.12 (Pygame compatible)

Framework: Pygame

Art Style: Pixel Art

Resolution: 1280x720

Target FPS: 60

🚀 Novidades Recentes (2025-11-09)

Sprites e Overworld (v0.9.3)

  • Player em batalha usa sprite Tiny Swords (Blue Warrior Idle).
  • Overworld: edifícios por relação (Blue/Black/Red/Yellow) — Castle, House1 e Tower (bandidos em dobro).
  • Ancoragem centrada no loc.pos para alinhar com a área de acesso (Press E).
  • Raio de interação/descoberta reduzido (castle≈0.65×radius; town/camp≈0.60×radius).
  • Sombra corrigida (alpha normal, sem quadrado preto).
  • Novos módulos: sprite_manager, animation, battle_sprites, world_sprites.

Facções e Mundo

  • Facções centralizadas em src/factions.py (loja/loot/spawn por facção).
  • Castelos por facção adicionados ao mapa (Macedon, Greeks, Ptolemaic, Seleucid, Rome, Carthage, Kush, Maurya, Pontus, Thrace).
  • Patrulhas periódicas partindo de cada castelo, com cap de densidade local.
  • Spawns iniciais e respawns ancorados em castelos (distribuição coerente por região).

Economia, Loja e Loot

  • Lojas de castelo vendem itens temáticos via get_random_shop_inventory(faction_id).
  • Loot pós-batalha por facção com factions.roll_loot(...).
  • Mais conteúdo 280 a.C.: Sarissa, Dory, Kopis/Xiphos, Gladius/Pilum/Scutum, Khopesh, Akinakes, Sabre, arcos compostos, Hamata/Linothorax/Lamelar/Escamas etc.

Combate e Correções

  • Ataques usam atributos da arma: range, cooldown, stamina_cost.
  • Números de dano não duplicam mais no mesmo swing (marcação por inimigo atingido).
  • Fix: Tooltip NameError, callbacks de equip/drop com Item, e atributos faltando nas patrulhas.

🏗️ Technical Architecture

📁 Project Structure

jogo rpg/
├── main.py              # Loop principal do jogo e gerenciamento de estados
├── requirements.txt     # Dependências do Python (Pygame)
├── install.bat          # Script para instalar dependências
├── run.bat              # Script para iniciar o jogo
├── README.md            # Informações gerais do projeto e como jogar
├── COMMUNICATION.md     # Canal de comunicação entre os desenvolvedores (IAs)
├── CHANGELOG.md         # Histórico de todas as mudanças e versões
├── ARCHITECTURE.md      # Documentação da arquitetura técnica do projeto
├── index.html           # Dashboard visual do projeto (abra no navegador)
│
└── src/                 # Código fonte principal do jogo
    ├── entities.py      # Definições de entidades (Player, Inimigos, Tropas) e seus atributos
    ├── rpg.py           # Lógica de progressão (XP, Nível, Dificuldade)
    ├── world.py         # Geração e gerenciamento do mundo, encontros e câmera
    ├── battle.py        # Sistema de combate tático em arena
    ├── equipment.py     # Definições e gerenciamento de equipamentos (armas, armaduras)
    ├── vfx.py           # Sistema de efeitos visuais e partículas
    ├── save_system.py   # Lógica para salvar e carregar o estado do jogo
    ├── items.py         # Definições de itens genéricos e seus tipos
    ├── factions.py      # Facções (loja/loot/spawn por facção centralizados)
    ├── item_bridge.py   # Funções de conversão e gerenciamento de itens
    ├── inventory_ui_v2.py # Interface de usuário para o inventário do jogador
    ├── ui_components.py # Componentes reutilizáveis de interface de usuário (botões, painéis, etc.)
    └── ui_shop.py       # Interface de usuário para a loja de equipamentos

⚙️ Core Systems

O jogo usa uma máquina de estados principal em `main.py` (`OVERWORLD`, `BATTLE`, `LOCATION_MENU`, `INVENTORY`, `SHOP_MENU`). As entidades são gerenciadas através de classes e dataclasses em `src/entities.py`, com a lógica de jogo distribuída em módulos especializados (`battle.py`, `world.py`, `rpg.py`).

For complete architecture documentation, see: ARCHITECTURE.md Includes: - Detailed system designs - Design patterns used - Coding conventions - Performance considerations - Debug features
